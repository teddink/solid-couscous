Introduction

In the first scenario, a variable is created for the MSGesture object. This object is used for the life of the webpage. When a user touches the screen element, a pointerdown event is fired and the pointer is assigned to the gesture object. While the user keeps contact, the element is moved around the screen as they slide and swipe their finger. The MSGestureChange event fires as the user interacts. The MSGestureChange event hander uses the element’s offsetX and offset Y properties to update the element’s CSS transform, effectively moving the element around the screen.

In the second scenario, we get the rotation and scale properties from the gesture object on the element. Those values are passed to the rotate and scale properties of the element’s CSS transform. This allows the element to reflect scale and rotation as the result of two finger pinches and turns. If the user tries to scale the element past a tenth of the original size, the scale property isn’t used.
